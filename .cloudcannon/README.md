## 1. First-time setup

You should have received an invite email from CloudCannon. Click it, set your password, then bookmark the app URL: `https://app.cloudcannon.com/`. From here forward you log in there.

**Lost access?** Email Cash at cash@thevoiceofcash.com or message David Jones (`coachdavidjones`) on Discord. We can reset your invite within an hour during agency hours.

If you ever want direct access to the image library (bulk uploads, asset cleanup), ask the agency to add you as a sub-user on the agency's Cloudinary account; we'll do it within 5 minutes.

## 2. The three places you'll work from

The left sidebar in CloudCannon has three sections that matter for day-to-day editing:

| Section | What's there | Editor used |
|---|---|---|
| **Pages (English — source of truth)** | Homepage, products, where-to-buy, about, etc. — every page on heatnglo.com in English | Visual Editor (drag-and-drop) |
| **Blog (English)** | Blog posts in Markdown | Content Editor |
| **Site Data** | `nav.json`, `footer.json`, `cta.json` — site-wide menu + footer + CTA configuration | Data Editor |

You'll see four other content groups in the sidebar — `Pages (Español — auto)`, `Pages (English-CA — auto)`, `Pages (Français-CA — auto)`, and the same for blog/services/faq. **Those are auto-generated by translation.** You don't edit them directly. See [§8 Multi-language](#8-multi-language-en--es--en-ca--fr-ca--how-it-works) for the full story.

## 3. How to edit safely

The 5-step save workflow:
1. Click into the page you want to edit (Pages → e.g. "Home").
2. Make your change. Text is inline-editable; clicking on highlighted regions activates them.
3. Watch the top-right status: **green dot = synced, yellow = saving, red = sync issue** (see [§12](#12-sync-errors--recovery)).
4. **Save Draft** keeps your change visible only in the CMS preview. **Publish** commits it to GitHub and pushes to the live site within 60–90 seconds.
5. After Publish, refresh `heatnglo.com/<the-page-you-edited>` in a private/incognito window. If the change is there, you're done. If not, see [§14 the 6-step debug ladder](#14-browser-requirements--the-6-step-debug-ladder).

The blue outline you'll sometimes see around editable elements means CloudCannon recognizes that field as inline-editable. Click it; type; the blue outline turns into a save indicator.

## 4. Adding or editing page sections

The homepage and most marketing pages use a **content-block system** — sections are reorderable, addable, removable. To use it:

1. Click into a page in the Visual Editor.
2. Each section has a small drag handle on its left edge — hover to see it.
3. **Drag** to reorder.
4. Click the **+** button between two sections to add a new section. You'll see a picker with options (Hero, Trust Strip, Love Your Fireplace Again, Browse the Lineup, ProAdvantage, Six TV Walls, Press Strip, FAQ, Owner Quote). Pick one; it inserts pre-filled with sample copy you replace.
5. Click the **menu icon** on a section to delete or duplicate.

Inside each section, every visible piece of text + every image is editable in place. Headings, body paragraphs, button labels — click and edit.

## 5. Adding images via Cloudinary

Heat & Glo's image library lives on Cloudinary at `voc-agency` cloud. Inside CloudCannon, every image field shows two buttons:
- **Upload** — drops a new image into the library and selects it for this field
- **Choose** — picks from the existing library

**Image-size quick reference:**

| Use case | Recommended dimensions | File format |
|---|---|---|
| Hero background (full-bleed) | 1920 × 1080 | JPG (faster) or WebP |
| Section image (Love Your Fireplace, ProAdvantage, TV Walls) | 1000 × 750 | JPG |
| Lineup card thumbnail | 600 × 450 | JPG |
| OG / social-share image | 1200 × 630 | JPG |
| Blog hero | 1600 × 900 | JPG |
| Logo / wordmark | (use the SVG we ship) | SVG |

**Alt text matters.** Every image input has an "Alt text" field next to it — fill it in. Screen readers read it; Google indexes it; visitors with slow connections see it before the image loads.

**Don't delete images that might be in use.** Cloudinary's "Used In" tab shows you whether an image is referenced by any page. If you're not sure, ask the agency before deleting.

## 6. Adding a blog post

1. Sidebar → **Blog (English)** → **+ New Blog Post**.
2. Fill in: Title, Description (1-line summary), Author, Date, Category, Tags, Hero image, Body.
3. Body is **Markdown** — see [§15 glossary](#15-glossary) for syntax. Bold, italics, headings, bullet lists, links — all standard.
4. **Save Draft** while you're writing; **Publish** when ready. Drafts don't appear on the live site.
5. Once published, the post lands at `heatnglo.com/blog/<slug>/`.

## 7. SEO fields on every page

Every page has three SEO fields at the top:
- **Title** — what shows in browser tabs + Google search results. Keep under 60 characters.
- **Description** — the meta description Google shows under the title. Keep under 160 characters. Include the primary keyword + a value-prop.
- **OG Image** — the image Facebook / X / LinkedIn show when someone shares the page. 1200 × 630 ideal.

If you don't fill these in, Heat & Glo's SEO suffers. They're not optional.

## 8. Multi-language (EN / ES / EN-CA / FR-CA) — how it works

Heat & Glo ships in 4 locales:
- `heatnglo.com/` — US English (default)
- `heatnglo.com/es/` — US Spanish
- `heatnglo.com/en-ca/` — English Canada
- `heatnglo.com/fr-ca/` — Quebec French

**You only edit English.** Every other locale is auto-translated from your English content via Anthropic (Claude). The translation runs every time you publish:
- Spanish translation = warm marketing tone, US Spanish (not Spain Spanish, not Mexico-only Spanish — broad-Hispanic-American).
- Canadian English = light localization (ZIP → postal code, $X → $X CAD, color → colour).
- Quebec French = Quebec-specific French (NOT Parisian) — `courriel` not `e-mail`, `magasinage` not `shopping`, `foyer` not `cheminée`.

You'll see four locale groupings in the sidebar — **only the English-source group is editable.** The others show what's been translated; if a translation looks wrong and you want to override it, ask the agency to mark that string as locked.

## 10. Draft → Publish workflow

- **Save Draft** — visible only inside CloudCannon's preview. Live site doesn't change. Use this when you're mid-edit and want to come back later.
- **Publish** — commits to GitHub, triggers a Vercel rebuild (60–90 seconds), live site updates. Your edit is now public.

When in doubt, draft first. Drafts don't pressure you to ship before you're ready.

## 11. Version history & undo

Top-right of any page editor → click the clock icon → see version history. Every save is a separate version. Hover any version to preview; click to restore.

History goes back 90 days. Older than that, the agency has Git history; we can recover anything.

If you accidentally delete a whole post or page, **don't panic.** Tell the agency immediately; we restore it from Git in under 5 minutes.

## 12. Sync errors & recovery

The colored dot in the top-right of CloudCannon tells you sync status:
- **Green** = synced. Everything's fine.
- **Yellow** = saving in progress.
- **Red** = sync issue. Don't keep editing until it's resolved.

If you see red:
1. Hover the dot for the error message.
2. Three recovery options usually appear:
   - **Reset to remote** — discards local-only changes, pulls from GitHub. Use when GitHub has the right state and you're seeing weird stale-data issues.
   - **Start a new branch** — preserves your in-progress changes in a side-branch the agency can review/merge. Use when you've made hours of edits you don't want to lose.
   - **Switch to another branch** — useful only if you have multiple branches active.
3. If unsure, screenshot the error and ping the agency on Discord. Don't guess — sync errors compound if mishandled.

## 14. Browser requirements + the 6-step debug ladder

**Supported:** Chrome, Edge, Safari, Firefox (latest 2 versions of each). Brave works but Shields can block parts of the Visual Editor — disable Shields for `app.cloudcannon.com`.

**When the CMS seems broken but isn't, try this in order:**
1. **Hard refresh** — Cmd+Shift+R (Mac) / Ctrl+Shift+R (Windows). Cleans browser cache.
2. **Logout + login** — fixes 80% of "I can't see X" issues.
3. **Incognito window** — rules out browser extension interference.
4. **Different browser** — Chrome → Safari, see if the issue follows.
5. **Wait 2 minutes** — Vercel deploys take 60–90 seconds; sometimes you're looking before the build's done.
6. **Check the sync dot** — if it's red, see [§12](#12-sync-errors--recovery).

If all 6 fail, **then** contact the agency.

## 15. Glossary

- **CMS** — Content Management System. CloudCannon is a CMS.
- **DAM** — Digital Asset Manager. Cloudinary is your DAM.
- **CDN** — Content Delivery Network. Vercel + Cloudinary both use CDNs to serve fast.
- **Markdown** — a plain-text formatting syntax used in blog post bodies. `**bold**`, `*italic*`, `[link](url)`, `# Heading`, `- bullet`. GitHub-flavored Markdown (GFM) is the dialect we use.
- **Frontmatter** — the YAML block at the top of a Markdown post (title, description, date, etc.).
- **Slug** — the URL-safe version of a title, e.g. "Six TV Wall Designs" → `six-tv-wall-designs`.
- **Draft / Publish** — Draft = saved but not live; Publish = live on the site.
- **Commit** — a Git save-point. Every Publish creates a commit.
- **Branch** — a parallel version of the site. We use `main` (production) primarily.
- **Repository (Repo)** — the GitHub project that holds the site's code + content.
- **Astro** — the framework that builds the site. You don't interact with it directly.
- **Vercel** — the hosting platform. Auto-deploys whenever you Publish.
- **Pagefind** — the search engine that powers in-site search.
- **Cloudinary** — the image hosting + optimization service.
- **OG Image** — Open Graph image. The preview image shown when someone shares a page on Facebook / X / LinkedIn.
- **GFM** — GitHub-Flavored Markdown. Specific Markdown dialect with table support, task lists, etc.
